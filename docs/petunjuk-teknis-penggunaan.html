Penggunaan Website Procurement Workflow

Panduan operasional untuk requester dan procurement admin dalam melakukan pengajuan, unggah dokumen, persetujuan tahap, pengaturan workflow, dan pengelolaan akun.

1. Tujuan dan Ruang Lingkup

Website Procurement Workflow digunakan untuk mengelola proses pengadaan secara bertahap. Proses disusun menjadi Level 1, Level 2, dan Level 3. Level 1 adalah aktivitas utama, Level 2 adalah sub-aktivitas, sedangkan Level 3 adalah dokumen yang wajib diunggah oleh requester.

Prinsip alur: requester hanya dapat mengirim satu Level 1 jika seluruh dokumen Level 3 pada Level 1 tersebut sudah diisi. Setelah dikirim, requester menunggu persetujuan admin sebelum dapat lanjut ke Level 1 berikutnya.

2. Peran Pengguna

Peran Fungsi Utama
Requester Membuat entry pengadaan, mengunggah dokumen wajib, melihat status tiap Level 1, dan mengirim pengajuan.
Procurement Admin Melihat seluruh entry, menyetujui atau menolak submission, mengatur workflow, dan mengelola akun pengguna.

3. Login dan Logout

Login

  1. Buka website melalui browser.
  2. Pilih peran login: Requester atau Procurement Admin.
  3. Masukkan email dan password.
  4. Klik tombol login untuk masuk ke dashboard sesuai peran.

Logout

  1. Klik tombol Logout pada bagian atas aplikasi.
  2. Sistem akan menghapus sesi login dan kembali ke halaman login.
Jangan membagikan akun admin kepada pengguna lain. Gunakan halaman Accounts untuk membuat akun baru jika ada pengguna yang membutuhkan akses.

4. Penggunaan sebagai Requester

Membuat Entry Pengadaan

  1. Login sebagai requester.
  2. Pada bagian entry, isi judul atau nama pengadaan.
  3. Klik tombol untuk membuat entry.
  4. Pilih entry yang ingin dikerjakan apabila terdapat lebih dari satu entry.

Mengisi Dokumen Level 3

  1. Lihat status Level 1 pada bagian atas form.
  2. Buka Level 1 yang sedang berstatus In progress.
  3. Untuk setiap Level 2, unggah dokumen pada field Level 3 yang tersedia.
  4. Pastikan seluruh field dokumen wajib sudah terisi.

Mengirim Submission

  1. Setelah semua dokumen wajib diunggah, klik tombol Submit.
  2. Sistem akan memvalidasi kelengkapan dokumen.
  3. Jika ada dokumen kosong, sistem menampilkan daftar dokumen yang masih harus diunggah.
  4. Jika berhasil, status Level 1 menjadi menunggu approval admin.

5. Penggunaan sebagai Procurement Admin

Dashboard

Dashboard admin menampilkan daftar entry pengadaan dari requester. Admin dapat melihat posisi Level 1 saat ini, status approval, dan detail dokumen yang sudah diunggah.

  1. Login sebagai procurement admin.
  2. Buka tab Dashboard.
  3. Pilih entry yang ingin diperiksa.
  4. Lihat dokumen yang diunggah requester.
  5. Setujui submission jika dokumen sudah sesuai, atau tolak jika perlu revisi.

Status Entry

Status Arti
Not started Level 1 belum dapat dikerjakan atau belum dimulai.
In progress Requester sedang mengisi dokumen pada Level 1 tersebut.
Pending approval Requester sudah submit dan menunggu keputusan admin.
Approved Admin sudah menyetujui Level 1 dan requester dapat lanjut ke Level 1 berikutnya.
Rejected Submission ditolak dan requester perlu memperbaiki dokumen.

6. Workflow Builder

Workflow Builder digunakan admin untuk mengatur susunan proses pengadaan. Admin dapat menambah, menghapus, mengubah nama, mengubah deskripsi, dan mengatur urutan Level 1, Level 2, serta Level 3.

Mengatur Level 1

  1. Buka tab Workflow Builder.
  2. Gunakan panel kiri untuk memilih Level 1.
  3. Gunakan tombol Add Level 1 untuk menambah aktivitas utama.
  4. Gunakan tombol Up atau Down untuk mengubah urutan.
  5. Klik Save Changes di bagian atas halaman untuk menyimpan perubahan.

Mengatur Level 2

  1. Pilih Level 1 yang akan diedit.
  2. Klik Add Level 2 untuk menambah sub-aktivitas.
  3. Isi nama dan deskripsi Level 2.
  4. Gunakan tombol hide/show untuk melipat atau membuka isi Level 2.
  5. Klik Save Changes setelah selesai.

Mengatur Level 3

  1. Pada Level 2 yang dipilih, klik Add Level 3.
  2. Isi nama dokumen yang wajib diunggah requester.
  3. Susun urutan dokumen sesuai kebutuhan proses.
  4. Klik Save Changes agar perubahan tersimpan.
Perubahan di Workflow Builder tidak otomatis tersimpan. Gunakan tombol Save Changes setelah selesai mengedit agar perubahan tidak hilang saat halaman dimuat ulang.

7. Manajemen Akun

Halaman Accounts digunakan admin untuk melihat dan mengelola seluruh akun yang terdaftar.

Fitur yang Tersedia

  • Melihat daftar akun requester dan procurement admin.
  • Membuat akun baru dengan password sementara.
  • Mengubah nama dan email akun.
  • Mengangkat requester menjadi procurement admin.
  • Menurunkan admin menjadi requester.
  • Mengaktifkan atau menonaktifkan akun.
  • Mengganti password akun jika diperlukan.
Admin yang sedang login tidak dapat menonaktifkan atau menurunkan role akunnya sendiri. Proteksi ini mencegah admin kehilangan akses secara tidak sengaja.

8. Struktur Penyimpanan Dokumen

Dokumen yang diunggah disimpan secara terstruktur sesuai hierarki workflow. Struktur folder mengikuti pola:

folder_level1s / folder_level2s / folder_level3s / file_dokumen

Contoh:

data/documents/01_perencanaan/01_requirement-definition/01_terms-of-reference/dokumen.pdf

Struktur ini memudahkan pencarian dokumen berdasarkan aktivitas utama, sub-aktivitas, dan jenis dokumen.

9. Validasi dan Status

Validasi Requester

  • Semua field dokumen wajib harus diisi sebelum submission dapat dikirim.
  • Jika masih ada field kosong, sistem menampilkan pesan dokumen mana yang belum diunggah.
  • Requester tidak dapat lanjut ke Level 1 berikutnya sebelum Level 1 sebelumnya disetujui admin.

Validasi Admin

  • Hanya akun dengan role procurement admin yang dapat membuka Dashboard admin, Workflow Builder, dan Accounts.
  • Akun yang dinonaktifkan tidak dapat login.
  • Perubahan role dan status akun dilakukan dari halaman Accounts.

10. Troubleshooting Singkat

Masalah Solusi
Tombol submit tidak berhasil Pastikan semua dokumen Level 3 pada Level 1 aktif sudah diunggah.
Tidak bisa login Periksa email, password, role login, dan pastikan akun masih aktif.
Perubahan workflow tidak tersimpan Klik tombol Save Changes setelah mengedit workflow.
Port 3000 sudah digunakan Tutup proses server lama atau jalankan aplikasi pada port lain.
Ngrok tidak dikenali Gunakan path lengkap ngrok/ngrok.exe http 3000 atau tambahkan ngrok ke PATH Windows.

Dokumen ini dapat diperbarui jika alur aplikasi, role pengguna, atau kebijakan dokumen berubah.
